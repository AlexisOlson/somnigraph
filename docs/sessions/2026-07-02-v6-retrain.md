  `tune_gt.load_data` loads `vector_input_map` from `recall_meta`; sidecar overrides

## Two code fixes the run forced (both committed)

1. **`49d290e` — OpenAI embed batch cap.** `precompute_searches` sent all 3,335
   missing query texts in one `embeddings.create` call; OpenAI rejects >2,048 inputs.
   Never bit before because prior runs grew the embed cache incrementally.
   `_openai_embed_batch` now chunks at 1,000 with per-chunk retry.
2. **`400f6f0` — CV memory hygiene.** At 4,883,191 rows × 31 features, each fold's
   `X[train_idx]` fancy-index copy is ~460MB, and the previous fold's copies are still
   referenced while the next fold's slice is evaluated — fold 1 OOM'd. Per-fold
   `del` + `gc.collect()` in both CV loops, and the `--train-only` pickle load now
   streams instead of holding raw bytes + objects simultaneously (~2× peak). The
   rerun used `--train-only` (extraction, embed cache, and PPR cache had all
   survived the OOM'd run on disk).

Reload identity was asserted before interpreting any gate: 4,883,191 × 31, 4,316
queries, and the full per-row weight distribution matched the interrupted extraction
run exactly (0.0: 305,766 / 0.5: 1,307,740 / 1.0: 2,656,764 / 1.2: 610,957 /
2.5: 1,155 / 5.0: 269 / 6.0: 540).

## Task 4 — March GT continuity slice (both cuts)

`gt_calibrated.json` (1,032q, March 20) restricted to still-active memories:
1,028 queries survive (96.5% of labels). 884 of them also appear in the fresh V6
GT, so the full slice is flagged in-sample; the 144-query disjoint subset is the
only genuinely out-of-sample continuity point (noisy at that n).

| Cut | n | V6 NDCG@5k | Formula NDCG@5k | Delta |
|---|---|---|---|---|
| Full slice (in-sample-flagged) | 1,028 | 0.7194 | 0.7063 | **+0.0131** |
| Disjoint (out-of-sample) | 144 | 0.7025 | 0.6916 | **+0.0108** |

References on this instrument (March-era corpus): formula ~0.74, March 26-feature
model ~0.81. Reading: **V6 beats today's formula on both cuts, but the March-era
+7pp learned-vs-formula gap does not carry over on this instrument — and the
instrument itself has drifted.** Today's formula reads 0.7063 vs its March-era
~0.74 on the same GT family, and March queries now face ~400 post-March memories
as unlabeled implicit negatives, which structurally punishes a learned ranker
(trained to surface exactly the memories the March GT never saw) more than the
formula. The in-sample overlap barely inflates (full vs disjoint: +1.7pp V6,
+1.5pp formula — labels differ even where queries overlap). Reported as-is;
the March slice was a continuity instrument, not a gate. Note also: absolute R@10
on this instrument (~0.25–0.29 for both scorers) is not comparable to fresh-GT
R@10 — the March GT averages ~65 graded labels/query.

The optional March-GT training merge was **declined**: candidate pools come from
live search on the current DB, and there is no clean per-query time-scoping in the
extraction pipeline — post-March memories would land as implicit negatives for
March queries. V6 is trained fresh-only.

  under ~3 months of **formula-ranked** retrieval (the fallback period) — the same
